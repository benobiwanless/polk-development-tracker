# Polk Development Tracker

## Version 6: Auto-Fetch Build

This version adds visible live-source cards and a GitHub Actions workflow.

### What changed

- New homepage section: "Latest Pulled Updates"
- New `data/live-news.json`
- New `data/last-updated.json` generated by workflow
- New GitHub Actions workflow: `.github/workflows/fetch-live-data.yml`
- New Python fetch script: `tools/fetch_sources.py`
- New automation instructions page

### How to activate live pulling

1. Upload all files/folders to the root of your GitHub repo.
2. Make sure the `.github/workflows/fetch-live-data.yml` file is uploaded.
3. Go to GitHub → your repo → Actions.
4. Choose "Fetch Live Data".
5. Click "Run workflow".
6. Wait for it to complete.
7. Refresh the GitHub Pages site.

The workflow also runs every 6 hours.

### Important

GitHub Pages alone cannot scrape live websites in the browser. GitHub Actions does the fetching, saves results into JSON, then GitHub Pages displays that JSON.

Facebook pages are not directly scraped. Use them as manual lead sources.
